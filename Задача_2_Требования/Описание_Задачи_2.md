# Задача 2 – Формирование требований к новой функциональности

## Описание задания

В рамках задания необходимо сформулировать требования для реализации новой функциональности личного кабинета продавца.

Новая функциональность должна позволять продавцу публиковать свой товар на маркетплейсе (витрине товаров).

---

## Цель

Описать функциональность публикации товара с использованием следующих артефактов:

- User Story
- Use Case
- Диаграмма Use Case

---

## Требования

1. Выполнить верхнеуровневую постановку в формате User Story

2. Описать требования в формате Use Case

3. Опционально отразить процесс публикации товара в виде диаграммы

---

## Основные участники процесса

### Основной актор
- Продавец

### Система
- Личный кабинет продавца
- Витрина маркетплейса

---

## Основные сценарии

### Основной сценарий
1. Открытие раздела "Товары"
2. Создание карточки
3. Заполнение данных
4. Загрузка изображения
5. Проверка обязательных полей
6. Публикация товара
7. Отображение на витрине

---

### Альтернативный сценарий
1. Ошибка валидации
2. Возврат к заполнению формы

---

## Используемые инструменты

- Miro
- UML Use Case Diagram
- Activity Diagram
