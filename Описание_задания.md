# Описание тестового задания

## Задача 1

Сделайте модель в нотации BPMN 2.0 по описанию рабочего потока ниже.

---

### Описание

В компании «X» работает выдача IT-оборудования пользователям по заявке, которую надо оформлять в системе №1.

Пользователь офиса, когда ему нужно, делает заявку через почту или по телефону в 1-ую линию технической поддержки на оборудование.

Он получает номер обращения, благодаря которому может отследить, что происходит с его заявкой.

Первая линия технической поддержки делает заявку на оборудование в системе №2 отделу снабжения.

Отдел снабжения принимает заявку в системе №2.

Далее отдел снабжения проверяет наличие оборудования в системе №3.

Если оборудование есть — снабжение вызывает логистику и отправляет оборудование.

Если оборудования нет — делает заявку поставщику, принимает поставку и передает пользователю.

## Основные участники процесса

- 1-я линия технической поддержки

- Логистика
- Поставщик
- Система №1
- Система №2
- Система №3

---

## Основные сценарии

### 1. Оборудование есть в наличии
- пользователь создает обращение
- поддержка регистрирует заявку
- снабжение проверяет наличие
- логистика передает оборудование
- поддержка выдает пользователю
- заявки закрываются

### 2. Оборудования нет
- снабжение создает заявку поставщику
- ожидание поставки
- прием оборудования
- передача через логистику
- выдача пользователю
- закрытие заявок
